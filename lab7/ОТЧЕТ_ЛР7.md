## 2. Задание

Создать систему логирования для долгосрочного хранения данных (MongoDB). Итоговый класс должен содержать **как минимум два различных метода** для сохранения данных в **разные коллекции**.

## 3. Код классов и приложения

### 3.1. Класс логирования MongoDB (`things.py`)

В проект добавлен класс `Logger`, который:

- создаёт подключение `pymongo.MongoClient('mongodb://localhost:27017/')`;
- хранит **последние записанные значения** для исключения дублей;
- содержит минимум два метода записи в разные коллекции:
  - `insert_climate_reading()` → `ClimateReadings`
  - `insert_soil_reading()` → `SoilReadings`
  - `insert_valve_state()` → `ValveStates`

Файл: `c:\Users\admin\PycharmProjects\PythonProject\things.py`

### 3.2. Использование логгера в Flask (`app.py`)

В `app.py` создаётся объект `_logger = things.Logger("IOT_logger_db")` и вызываются методы логирования в маршрутах мониторинга:

- `GET /connect/climate` → `_logger.insert_climate_reading(...)`
- `GET /connect/soil` → `_logger.insert_soil_reading(...)` + `_logger.insert_valve_state(...)`
- `GET /connect/valve` → `_logger.insert_valve_state(...)`

Файл: `c:\Users\admin\PycharmProjects\PythonProject\app.py`

### 3.3. Зависимости

В `requirements.txt` добавлено:

- `pymongo`
